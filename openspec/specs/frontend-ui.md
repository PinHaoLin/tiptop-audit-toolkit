# Frontend UI Specification

## Delivery Prompt

交付提示只存在於左側主工作區，右側執行環境欄位不顯示重複提示。提示卡依內容高度呈現，不得因雙欄等高規則被拉伸。

## Unified Width System

頁面所有主要區塊共用同一個最大內容寬度。Grid 子項目、Card 與文字容器必須可以縮小，任何長路徑或提示文字不得使父層突破內容邊界。

## Responsive Workflow Layout

流程頁面在桌面寬度使用主工作區與執行環境側欄的雙欄布局，在窄視窗改為上下堆疊。所有 Grid 子項目必須可以縮小，不得由長文字內容撐開頁面。

## Path Display

稽核報告、執行根目錄、aud、暫存資料夾與本週資料夾均透過共用 `PathLine` 顯示。路徑文字不得超過父層寬度；長內容應在自身區域內以單行水平捲動方式呈現。

## Step 7 Report Review

步驟 7 顯示稽核報告路徑供人工確認，該路徑僅作為顯示資訊，不得改變報告檔案的實際產出位置或後端流程。
