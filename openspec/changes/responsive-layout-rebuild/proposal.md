# Responsive Layout Rebuild

## Why

目前頁面在部分視窗尺寸下，主內容 Grid 會被子元件的最小內容寬度撐開，造成上方標題與流程元件右側出現不必要空白，並讓主卡與執行環境卡的邊界不一致。

## What Changes

- 重整頁面外層容器，使標題、流程、提示與工作區共用同一個最大內容寬度。
- 讓所有 Grid、Card、Alert、Descriptions 與路徑節點可縮小，不得以內容寬度突破父層。
- 定義桌面、平板與手機三段布局，確保工作區不產生水平溢位。
- 保留既有流程功能、步驟切換、後端資料與交付提示內容。

## Scope

本次只重整前端布局與 CSS，不變更 Python 後端稽核流程、資料夾定位、報告產出或 EXE 資料處理行為。
